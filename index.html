01 — Overview

What This Project Does

This project turns a microcontroller into an intelligent environmental station for any room. Using the DHT11 sensor and BH1750 light module, it continuously measures temperature, humidity, and ambient light — combining all readings into a single "Comfort Score" displayed on a 16×2 LCD.

Two sub-projects stack together: a Temperature & Humidity Station powered by the DHT11, and a Light Intensity Meter using the BH1750. Both feed into the Comfort Meter, which outputs a human-readable verdict every second.

The I²C bus keeps wiring clean — the LCD and BH1750 share the same two-wire SDA/SCL line, minimizing connections and making the circuit easy to debug.

02 — Hardware

Sensors & Modules

01 — TEMPERATURE & HUMIDITY
🌡
DHT11
Digital Humidity & Temperature Sensor
Reads ambient temperature and relative humidity via a calibrated digital signal. Single-wire protocol means minimal wiring — just one data pin plus power and ground.
Temperature: 0–50 °C (±2°C accuracy) Humidity: 20–90% RH (±5% accuracy) Sampling rate: 1 Hz (once per second) Interface: Single-wire digital
02 — LIGHT INTENSITY
☀
BH1750
Digital Light Intensity Module
Measures ambient light precisely in Lux — the same unit used in professional lighting design. Communicates over I²C, sharing the bus with the LCD for a clean 4-wire setup.
Range: 1–65,535 lx Resolution: 1 lx Interface: I²C (address 0x23 or 0x5C) No ADC converter needed
03 — DISPLAY
📟
I2C 1602 LCD
16×2 Character LCD with I²C Backpack
Standard 16×2 LCD fitted with an I²C backpack. Reduces wiring from 8 pins down to 4 (VCC, GND, SDA, SCL) while sharing the same bus as the BH1750.
Display: 16 × 2 characters Interface: I²C via PCF8574 expander Default address: 0x27 Adjustable contrast via onboard pot
03 — Project Idea

The Comfort Meter

The Comfort Meter combines all three sensor readings into one clear verdict, printed on the second row of the LCD. Every second, the microcontroller checks current conditions against preset thresholds and updates the display.

If temperature or humidity exceeds comfort limits, it shows ⚠ TOO HOT. If light falls below the healthy working level, it shows ◑ TOO DARK. Otherwise all conditions pass and it shows ★ PERFECT.

04 — Implementation

How It Works

STEP 01
Read DHT11
The microcontroller sends a start signal on the DHT11 data pin. The sensor responds with a 40-bit packet containing temperature and humidity, decoded and stored each cycle.
STEP 02
Read BH1750 via I²C
A measurement request is sent to the BH1750 over I²C. The sensor returns a 16-bit integer in raw counts, which is divided by 1.2 to produce the final Lux reading.
STEP 03
Apply Comfort Logic
All three values are checked against threshold constants in code. A priority check runs: Too Hot takes precedence over Too Dark. If neither triggers, the verdict is Perfect.
STEP 04
Update the LCD
The LCD is cleared and rewritten every second. Row 1 shows temperature, humidity, and Lux values. Row 2 shows the Comfort Meter verdict — ★ PERFECT, ⚠ TOO HOT, or ◑ TOO DARK.
